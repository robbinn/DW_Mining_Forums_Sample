WebRTC: Vulnerability or functionality?

  • Quote

Postby Bali » Tue Aug 21, 2018 12:41 pm

WebRTC is an API developed by the World Wide Web Consortium (W3C), which supports browser-to-browser video chat, voice communication and P2P file sharing applications without the need for external plug-ins. In May 2011, Google launched a project for a Web-based open-source real-time communication platform known as WebRTC. The W3C version is undergoing refining and conversion into Chrome and Firefox browsers.

According to Mozilla, WebRTC's ability to issue information through the visitor's web browser is not a vulnerability, but a functionality. For this purpose, browsers must comply with three requirements set out in a draft published to the IETF organization. Which is, however, the reason to worry and why is it classified as vulnerability by many people online?

What is the threat?
Although this feature may be useful to some users, it poses a threat to anyone who uses a proxy server (VPN or other) and seeks to maintain online anonymity without revealing its true IP address. "

Application
WebRTC can be used to reveal your real IP address as well as your local IP address. This is possible through STUN queries with Firefox, Chrome, and Opera browsers even when using VPN.

In short, this means that each site can send multiple specific queries via Javascript to get your real IP address using the web browser you are using.

Can VPN Services Ensure Complete Anonymity?
The answer to this question is just as difficult as the question itself - perhaps.

To be best informed, it is recommended to contact your VPN service provider to see if additional protection is available on their behalf to prevent leakage of information through WebRTC.

Solution of the problem
There are several solutions to the problem, some of which are:

1. Turn off the WebRTC functionality from the browser you are using.
2. Use external plug-ins and browser add-ons to block the vulnerability. Keep in mind that this solution is not always 100% secure.
3. Use a VPN that has a WebRTC protection policy. Some of them are - ExpressVPN, HoxxVPN, Perfect Privacy
4. Use Tor Browser, which has WebRTC turned off and some other protections implemented.

Since Google Chrome and its other browsers can not shut down WebRTC under Desktop, the only two protection options are the use of add-ons and the above-mentioned VPN services.

To exclude WebRTC from Mozilla Firefox, type about.config in the URL field and then type "media.peerconnection.enabled" in the search box. Double-click the preference to change the value to "false."

To disable WebRTC from the Chrome mobile version, enter the chrome://flags/#disable-webrtc field in the URL, then locate the WebRTC STUN origin header and disable it.

To turn WebRTC off in Opera, you will also have to use an external plug-in for Chrome, called WebRTC Leak Prevent. Then, in the advanced options for the WebRTC Leak Prevent extension, select "Disable non-proxied UDP (force proxy)," and then click on Apply settings.
