Postby sammyx » Tue Aug 21, 2018 9:11 pm

So, I recently tested some strange Microsoft Outlook plugins that exchanged huge SOAP XML messages with backend services. I needed a tool that would allow me to manipulate real-time queries and replace many `values' in the processing and fuzzy process.

I am lazy and I did not intend to go through all of them manually, so I needed a convenient solution that allowed for scripting, many of the parameters had to be interrelated, so it made fuzzing even slower.

My solution was mitmproxy (mitmproxy.org)

`pip3 install mitmproxy`

Code: Select all

mitm% tree
.
├── mitmproxy-ca-cert.pem
└── fuzz.py

0 directories, 2 files


where mitmproxy-ca-cert.pem has to be installed in the client, ie. added to trusted issuer authorities.
If you run mitmproxy and redirect your browser to redirect traffic to 127.0.0.1:8080 and go to mitm.it, you will be given the opportunity to directly install it.

image.thumb.png.b2e37b12a8af11a96fc56fb5ca2f4557.png (146.21 KiB) Viewed 22938 times


Burp is also a very functional tool, so I always put it as an upstream server and I set its sock proxy to invisible mode.

We start the server:

Code: Select all

mitm% mitmweb --mode upstream:127.0.0.1:8083 -k -s fuzz.py -v
Web server listening at http://127.0.0.1:8081/
Loading script fuzz.py


You can also not use the web interface and instead run it standard with "mitmproxy" and then the same parameters.
While mitmproxy is running, every time you Save/Ctrl-S on fuzz.py will cause mitmproxy to reload automatically, saving time for scripting and increasing fuzzy flexibility.
